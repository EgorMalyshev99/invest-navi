## Философия дизайна

### Главная идея

> «Личный инвестиционный помощник, который объясняет российский рынок простыми словами и помогает думать перед покупкой.»

### Визуальное позиционирование

- **Dark modern fintech** — тёмная, спокойная, уверенная палитра
- **Premium SaaS** — чистота, аккуратность, детализация
- **Clean editorial dashboard** — данные поданы как материал аналитического журнала
- **Не трейдинг-терминал** — никакого визуального давления на действие
- **Не крипто/gaming стиль** — без неоновых свечений, агрессивных анимаций, «мемных» элементов

### Принципы дизайна

| Принцип     | Описание                                                                     |
| ----------- | ---------------------------------------------------------------------------- |
| Спокойствие | Интерфейс не кричит и не подталкивает к импульсивным действиям               |
| Воздух      | Больше whitespace, меньше плотности данных на экран                          |
| Понятность  | Финансовая сложность выглядит доступно через визуальную иерархию             |
| Доверие     | Premium-ощущение: аккуратные карточки, консистентные отступы, читаемый текст |
| Доступность | Приятно и новичку, и более опытному пользователю                             |

## Цветовая палитра

Все цвета задаются в **hex**. Используется тёмная палитра.

### Темизация (Dark + Light)

- По умолчанию используется системная тема через `next-themes` (`defaultTheme="system"`, `enableSystem`).
- `:root` в `apps/web/app/globals.css` описывает `light` токены.
- Класс `.dark` в `apps/web/app/globals.css` описывает `dark` токены.
- Класс `.light` не используется.
- Компоненты **не** должны переключать классы вручную под каждую тему; использовать только семантические токены (`bg-background`, `text-foreground`, `border-border`, и т.д.).
- Акценты и продуктовая семантика сохраняются между темами: `primary`, `accent`, `positive`, `warning`, `negative`, `info`.

| Тема            | Основной фон | Основной текст | Surface   | Border    |

## Типографика

Базовая реализация: `apps/web/src/components/ui/typography.tsx` (`cva` + `variant` + `numeric` + `asChild`).

### Принципы типографики

- Заголовки: крупные, уверенные, `font-bold` / `font-semibold`
- Текст: читаемый (минимум 16px для body), комфортный `line-height`
- Не использовать «игровой», «криптовый» или слишком декоративный стиль
- Тон: спокойный, аналитический, понятный

### Шкала

| Элемент | Размер (Tailwind)       | Weight          | Line-height       |
| ------- | ----------------------- | --------------- | ----------------- |
| Display | `text-4xl` / `text-5xl` | `font-bold`     | `leading-tight`   |
| H1      | `text-3xl`              | `font-bold`     | `leading-tight`   |
| H2      | `text-2xl`              | `font-semibold` | `leading-snug`    |
| H3      | `text-xl`               | `font-semibold` | `leading-snug`    |
| H4      | `text-lg`               | `font-medium`   | `leading-normal`  |
| Body    | `text-base` (16px)      | `font-normal`   | `leading-relaxed` |
| Small   | `text-sm`               | `font-normal`   | `leading-normal`  |
| Caption | `text-xs`               | `font-medium`   | `leading-normal`  |

### Шрифт — Inter

**Основной шрифт:** [Inter](https://fonts.google.com/specimen/Inter)

| Свойство        | Значение                                                                  |
| --------------- | ------------------------------------------------------------------------- |
| Тип             | Sans-serif, variable font                                                 |
| Лицензия        | SIL Open Font License (бесплатный)                                        |
| Кириллица       | Полная поддержка                                                          |
| Латиница        | Полная поддержка                                                          |
| Tabular numbers | Есть (`font-variant-numeric: tabular-nums`) — все цифры одинаковой ширины |
| Subsets         | `latin`, `cyrillic`                                                       |

**Использование tabular numbers для финансовых данных:**

```css
.tabular-nums {
  font-variant-numeric: tabular-nums;
}
```

Применять `tabular-nums` на всех элементах с ценами, процентами, объёмами — чтобы цифры не «прыгали» при обновлении данных.

## Компоненты

### shadcn/ui — установленная библиотека

Конфигурация: `apps/web/components.json` (стиль `radix-lyra`, иконки `phosphor`).
Компоненты живут в `apps/web/src/components/ui/`.
Кастомизация — через CSS custom properties и Tailwind-классы.

#### Типографика (Typography)

Типографика реализуется через единый компонент `Typography` в `apps/web/src/components/ui/typography.tsx` на базе `cva`.

| API       | Описание                                                                                                         |
| --------- | ---------------------------------------------------------------------------------------------------------------- |
| `variant` | `display`, `h1`, `h2`, `h3`, `h4`, `body`, `lead`, `large`, `small`, `muted`, `blockquote`, `list`, `inlineCode` |
| `numeric` | `default` \| `tabular` (для финансовых чисел)                                                                    |
| `asChild` | Полиморфный рендер через `Slot`                                                                                  |

**Пример использования:**

```tsx
<Typography variant="h2">Картина рынка</Typography>
<Typography variant="muted">Обновлено 5 минут назад</Typography>
<Typography variant="body" numeric="tabular">+2.41%</Typography>
```

> **Правило:** не использовать сырой набор utility-классов в каждом месте отдельно. Для текстов использовать `Typography` и его `variant`-систему.

#### Формы (Forms)

**Подход:** `react-hook-form` + `zod` + shadcn Field/Form компоненты.

| Компонент    | Файл              | Назначение                                 |
| ------------ | ----------------- | ------------------------------------------ |
| `Field`      | `field.tsx`       | Обёртка поля формы (label + input + error) |
| `Input`      | `input.tsx`       | Текстовое поле                             |
| `InputGroup` | `input-group.tsx` | Группа input с prefix/suffix               |
| `Textarea`   | `textarea.tsx`    | Многострочный ввод                         |
| `Select`     | `select.tsx`      | Dropdown-выбор                             |
| `Checkbox`   | `checkbox.tsx`    | Чекбокс                                    |
| `RadioGroup` | `radio-group.tsx` | Группа radio-кнопок                        |
| `Switch`     | `switch.tsx`      | Toggle переключатель                       |
| `Slider`     | `slider.tsx`      | Ползунок (уровень уверенности)             |
| `Label`      | `label.tsx`       | Лейбл поля                                 |
| `DatePicker` | composition       | Popover + Calendar (react-day-picker)      |

**Правила форм:**

- Лейблы всегда visible (не только placeholder)
- Валидация через `zod` schema
- Ошибки отображаются под полем через `Field`
- DatePicker — композиция из `Popover` + `Calendar`, не отдельный компонент

**DatePicker — пример использования:**

```tsx
<Popover>
  <PopoverTrigger asChild>
    <Button variant="outline" data-empty={!date}>
      <CalendarBlank size={16} />
      {date ? format(date, 'PPP') : <span>Выберите дату</span>}
    </Button>
  </PopoverTrigger>
  <PopoverContent className="w-auto p-0">
    <Calendar mode="single" selected={date} onSelect={setDate} />
  </PopoverContent>
</Popover>
```

#### Раскрывающиеся секции (Accordion / Collapsible)

| Компонент     | Файл              | Назначение                                                |
| ------------- | ----------------- | --------------------------------------------------------- |
| `Accordion`   | `accordion.tsx`   | Вертикальные FAQ/списки с разворачиванием по секциям      |
| `Collapsible` | `collapsible.tsx` | Локальное раскрытие дополнительного контента внутри блока |

**Правила использования:**

- `Accordion` — когда есть список независимых или взаимоисключающих секций (например, FAQ по рискам, блоки «что это значит»).
- `Collapsible` — когда нужно мягко раскрыть детали **внутри** уже существующей карточки/панели, не меняя структуру экрана.
- По умолчанию контент должен быть свернут до компактного состояния, чтобы не перегружать dashboard.
- Триггеры раскрытия формулировать ясно: «Показать детали», «Что это значит», «Развернуть расчёт».

## Структура экранов

### 1. Главный dashboard — «Картина рынка»

```txt
┌─────────────────────────────────────────────────────┐
│  [Hero] AI-резюме дня: "Рынок сегодня спокойный..." │
├──────────────────┬──────────────────┬───────────────┤
│  Индекс Мосбиржи │  Курс рубля      │ Ключевая ставка│
├──────────────────┴──────────────────┴───────────────┤
│  Самые заметные: растущие ↑ / падающие ↓            │
├─────────────────────────────────────────────────────┤
│  Сектора: сильнее / слабее рынка                    │
├─────────────────────────────────────────────────────┤
│  "Что это значит простыми словами"                  │
└─────────────────────────────────────────────────────┘
```

**Визуальная задача:** пользователь за 5 секунд понимает: рынок спокойный, рискованный, растущий или снижающийся.

### 2. Watchlist — Список наблюдения

```txt
┌─────────────────────────────────────────────────────┐
│  Фильтры: тип | статус | период                    │
├─────────────────────────────────────────────────────┤
│  [Table / Card-list]                                │
```

## Адаптивность (Responsive)

### Breakpoints

| Breakpoint | Ширина           | Layout                       |
| ---------- | ---------------- | ---------------------------- |
| Mobile     | `< 768px`        | 1 column, bottom nav         |
| Tablet     | `768px – 1024px` | 2 columns, collapsed sidebar |

### Mobile

- Bottom navigation: Рынок / Watchlist / Дневник / Портфель / AI
- Карточки — одна колонка, полная ширина

- Графики упрощены (мини-sparklines)
- AI summary — выше сложных данных
- Compact metric cards (2 в ряд)

- 2-column grid для карточек
- Sidebar collapsed (иконки), expandable
- Таблицы сохраняются с горизонтальным scroll
- AI-панель — как overlay Sheet

### Desktop

- Full sidebar с иконками + текст
- Multi-column dashboard (3–4 колонки для metric cards)
- Большие аналитические карточки
- AI-помощник — sticky side panel справа (опционально)
- Breadcrumbs + заголовок страницы

## Семантика и доступность

> **Принцип:** базовый разумный подход, без фанатизма. Семантика не должна вставать выше shadcn/ui компонентов. Если shadcn/ui рендерит `<div>` — не оборачивать его в `<article>` ради формальной «правильности». Компоненты библиотеки уже содержат необходимые aria-атрибуты.

### Семантическая разметка (базовый уровень)

Используем семантические теги только на уровне layout-каркаса страницы:

```html
<header>
  — шапка приложения
  <nav>
    — sidebar, bottom nav
    <main>
      — основной контент страницы
      <aside>
        — AI-панель (если отдельным блоком)
        <footer>— подвал (если есть)</footer>
      </aside>
    </main>
  </nav>
</header>
```

Внутри `<main>` — обычные `<div>` + shadcn/ui компоненты. Не нужно вручную добавлять `<section>`, `<article>` на каждый блок — это создаёт лишний шум без реальной пользы.

### Accessibility (базовый уровень)

| Требование     | Реализация                                                 |
| -------------- | ---------------------------------------------------------- |
| Контраст       | Палитра подобрана с учётом WCAG AA                         |
| Focus states   | Оставлять дефолтные focus-visible от shadcn/ui             |
| Labels         | `aria-label` на иконках-кнопках без текста                 |
| Не только цвет | Positive/negative сопровождаются иконкой (↑/↓) или текстом |
| Формы          | `<label>` связан с `<input>` через `htmlFor`               |
| Кнопки         | Интерактивные элементы — `<button>`, не `<div onClick>`    |

**Не делать без необходимости:** live regions, сложные aria-роли, кастомные keyboard handlers — shadcn/ui уже реализует это внутри своих компонентов (Dialog, Select, Dropdown и т.д.).

## UX-принципы

### Пользователь не должен чувствовать давление

- Нет агрессивных CTA «Купи сейчас»
- Нет countdown-таймеров, FOMO-элементов
- Кнопки мягкие, не кричащие: `outline` или `ghost` варианты для вторичных действий
- Primary button используется только для одного ключевого действия на экране

### Прогрессивное раскрытие сложности

- Начальный вид — простой и понятный
- Дополнительные данные — за табами, expandable-секциями, tooltip'ами
- Финансовые термины сопровождаются `TermTooltip` при первом упоминании

### Основной путь пользователя

```txt
Понять картину рынка → Добавить активы в наблюдение →
Изучить карточку актива → Создать гипотезу в дневнике →
Вернуться позже и проверить результат
```

Интерфейс визуально поддерживает этот путь: от обзора к деталям, от наблюдения к осмысленному действию.

### Первое использование

- Empty states с понятным текстом и подсказкой что делать
- Мягкий onboarding: «Начни не с покупки, а со списка наблюдения»
- Не перегружать первый экран — постепенное знакомство с функционалом

## Анимации и переходы

### Принципы анимаций

- Минимализм: анимации не должны отвлекать
- Быстрые переходы: 150–200ms для hover, 200–300ms для layout shifts
- `ease-out` для появления, `ease-in` для исчезновения
- Skeleton loading вместо спиннеров

### Где использовать

- Page transitions — subtle fade (150ms)
- Card hover — мягкий подъём тени или border highlight
- Number changes — плавная смена значений (countup)
- Sidebar collapse/expand — slide transition (200ms)
- Skeleton → content — fade transition

### Где НЕ использовать

- Никаких «bouncing» или «pulsing» кнопок
- Без flashy-анимаций на финансовых данных
- Без автоматически проигрывающихся тяжёлых анимаций

## Иконки — Phosphor Icons

**Библиотека:** [`@phosphor-icons/react`](https://phosphoricons.com/)

| Свойство          | Значение                                  |
| ----------------- | ----------------------------------------- |
| Пакет             | `@phosphor-icons/react`                   |
| Количество иконок | 9000+ (6 весов на каждую)                 |
| Стили             | Thin, Light, Regular, Bold, Fill, Duotone |
| Tree-shaking      | Да — в бандл попадают только используемые |
| Лицензия          | MIT                                       |

**Почему Phosphor:**

- Больше вариантов весов (6 стилей) — flexible для разных контекстов
- Duotone-стиль хорошо вписывается в premium dark dashboard
- Широкий набор финансовых и аналитических иконок
- Консистентная сетка и оптический баланс
- Простой API: `<TrendUp size={20} weight="regular" />`

**Основной стиль:** `Regular` для UI-элементов, `Duotone` для акцентов и иллюстративных иконок, `Bold` для active state в навигации.

**Размеры:**

| Токен | Размер | Применение                                  |
| ----- | ------ | ------------------------------------------- |
| sm    | `16px` | Inline с текстом, бейджи, мелкие индикаторы |
| md    | `20px` | Кнопки, table cells, card headers           |
| lg    | `24px` | Навигация, заголовки секций                 |
| xl    | `32px` | Empty states, hero-блоки                    |

**Accessibility:**

- Иконки с текстом рядом: `aria-hidden="true"`
- Standalone иконки (без текста): обязательный `aria-label`

**Пример использования:**

```tsx
import { TrendUp, ChartLineUp, Wallet, Notebook } from '@phosphor-icons/react';

<TrendUp size={20} weight="regular" className="text-positive" />
<ChartLineUp size={24} weight="duotone" className="text-primary" />
```

| Уровень | Использование             | Стиль                                   |

## Реализация палитры в Tailwind v4

Значения hex задаются в [`apps/web/app/globals.css`](apps/web/app/globals.css): `:root` (light) и `.dark` (dark). Переключение темы — через `next-themes` (`attribute="class"`, `defaultTheme="system"`). Мост в Tailwind — `@theme inline` (shadcn + проектные токены). Шрифт — `@theme { --font-sans }`.

```html
<div class="bg-background text-foreground border-border">
  <h1 class="text-foreground">Заголовок</h1>
  <p class="text-secondary-foreground">Описание</p>
  <span class="text-positive tabular-nums">+2.4% ↑</span>
  <span class="text-negative tabular-nums">−1.1% ↓</span>
  <span class="text-primary">Акцент</span>
</div>
```

Проектные токены без аналога в shadcn: `surface-elevated`, `disabled`, `positive`, `warning`, `negative`, `info`, `risk-high`.
